# Formatting & Syntax

Enforced via `@stylistic/eslint-plugin` and EditorConfig.

## Rules

| Rule | Value |
|------|-------|
| Indentation | 4 spaces (no tabs) |
| Semicolons | Always |
| Quotes | Single quotes everywhere. Double quotes only in JSX attributes if needed. |
| Trailing commas | Always where valid (ES5+) |
| Max line width | 120 characters |
| Line endings | LF (`\n`) |
| End of file | Single newline |

## What's Not Enforced

Formatting opinions beyond the table above (JSX wrapping, object break style, arrow function parens) are left to author discretion. The goal is consistency on the things that cause merge noise, not micromanaging every line break.
